Assignment 1

10/9/2016

The most challenging aspect of this assignment was understanding the difference between block and inline elements. I had problems styling the <li> elements because I didn't realize that they were inline. When I added horizontal padding or margins to these elements, it did not affect the containing <ul> element. I solved this problem by creating a minimal example, and asking my knowledgable classmates on Piazza, who were able to quickly identify the problem.

The most incredible thing I learned from the assignment was how to make the round buttons, like the "add note" button. If I had seen a button like that before, I would have thought that it was an image. The fact that you can make a button like that with pure CSS opened my eyes to its power. I now wonder what other elements around the web are done in pure CSS. Could I reproduce Google's "waiting" animation with CSS? There were some elements of the assignment that I already knew; doing the CSS hover rules, for example, was old hat.

As always, w3schools.com was an amazing resource. Additionally, my classmates on Piazza were very helpful as well. When using Piazza, I always tried to create a minimal example of my problem and to explain the exact problem I was having, what I tried, and what the precise error was.

Two things I had to Google to complete the assignment were how to:

  1. create a round button in pure CSS
  2. use flex boxes
The only reason I had to Google these items, though, is because I tried to complete the assignment before the lecture on these topics.

What I learned doing Assignment 0

9/22/2016

I've done a little bit of HTML before. So, in order to enhance by own learning, I gave myself two extra objectives when doing this assignment:

  1. Use only HTML5 compliant code.
  2. Make the site look nice without using CSS (because CSS isn't allowed yet).

These seemed like some very simple objectives to begin with, but most of the HTML that I learned before (in the HTML4 days) is not compatible with HTML5! This makes sense, because HTML5 was designed to use with CSS, but without CSS my hands are tied in so many ways. For example, there is a huge white space between the title of this post and the date, but without CSS, I can't adjust the style of the header. Well, maybe I can use the <font> tag instead of the <h2> tag so it doesn't put a huge vertical whitespace underneath it, right? Nope, <font> doesn't exist in HTML5. I can't even change the background without using CSS!

Overall, I really came to appreciate how much was removed in the transition between HTML4 and HTML5. This makes sense, because so much functionality in HTML5 is supposed to be handled by CSS, but I never quite fully grasped the scope of the changes before now.

I used w3schools.com as a reference as I worked on this assignment. Mostly, I used it to figure out which tag attributes are no longer supported in HTML5 (turns out, a lot). I use it because it is a clear, easy to use website. Also, it always comes up first or second when doing Google searches for any basic web related things!
